Tile
  id
  type

Space - an enclosure space
  enclosure ID
  position



PlacedTile
  tile
  space
  did it complete the enclosure
  optional money delta

Offspring - a created / birthed tile
  child PlacedTile
  mother tile
  father tile

Destination - poorly named, more of a "placement"
  space
  optional Offspring
  optional money delta

Delivery - a delivered tile
  truck position
  tile
  optional Destination


----
So, some thoughts.

Tile and Space are pretty solid.

Though the serialized form of Space as an int is compact, that may no longer be needed, and maybe it's better to be explicit? Or have the int representation used everywhere?

The money deltas everywhere are maybe OK? But maybe we don't need to show bonuses until committed. In which case, the only thing we have are action costs (and the 1 coin going to original owner for the purchase).  So PossiblePurchase needs to come with a money delta.  Or -- gasp -- we re-implement that logic on the FE. Or, simply, don't show the money changes until after (except for the cost to the active player). Hmmm .....

Offspring is good in terms of pairing child and parents (good to match the parents with the child on the UI). However, it doesn't need to be a PlacedTile. I mean, it can be, but ... why?

The rest are sort of sloppily distinguished, serving similar things.

We have Delivery -- used for delivering tiles from trucks. It's not great to use to "deliver" coins, which aren't technically delivered, but could be construed. Making the destination optional means a lot of non-null assertions. Maybe it's better to keep the coins separate.

Delivery groups a truck position with a tile and a Destination. But it could just pair a truck position with a PlacedTile. And PlacedTile could grow an Offspring. And Offspring could then reasonably *be* a PlacedTile.


So maybe we should have:

Tile
  id
  type

Space - an enclosure space
  enclosure ID
  position

PlacedTile
  tile
  space: Space
  did it complete the enclosure
  optional money delta
  optional Offspring -- always null if this is a Offspring child

Offspring - a created / birthed tile
  child PlacedTile
  mother tile
  father tile

Delivery - a delivered tile
  truck position
  PlacedTile


So a "Destination" could be replaced by a PlacedTile, if it had an optional offspring.
